# Клиент-серверное приложение "Автосалон"
**Вариант 5 (3 звездочки)**

## Описание реализации

Проект реализует клиент-серверную архитектуру с использованием сокетов AF_INET (TCP) и многопоточной обработкой клиентов на стороне сервера.

### Модель взаимодействия
Использована **модель с межклиентским взаимодействием**:
- Сервер в бесконечном цикле принимает подключения от клиентов
- Каждый клиент обслуживается в отдельном потоке
- Сервер хранит базу данных автомобилей и рассылает уведомления всем клиентам о событиях (продажа/поступление машин)

## Структура проекта

Программа состоит из следующих модулей:
- `protocol.h` - общий заголовочный файл с протоколом обмена
- `server.c` - исходный код сервера
- `client.c` - исходный код клиента

## Запуск программы

### Запуск сервера

./server <порт>     (порт по умолчанию 7777)

## Запуск клиента

./client <ip_сервера> <порт>
./client 127.0.0.1 7777

## Работа с клиентом

После подключения доступны следующие команды:
    • BUY <марка> - Купить машину
    • SELL <марка> - Продать машину
    • CARS - Показать список машин в наличии
    • exit - Выход из программы

## Valgrind

Запуск сервера: valgrind --leak-check=full ./server 7777

## server.c (Сервер автосалона)

Слушает порт, принимает клиентов и управляет базой машин.
Сервер может обслуживать до 1000 клиентов одновременно, база данных может хранить до 1000 машин

### Структуры данных

- `Car`: Простая структура, хранит только brand (название марки) машины.
    
- `Client`: Структура для хранения информации о подключенном клиенте.

   ◦ `socket`: файловый дескриптор сокета (число, по которому мы обращаемся к клиенту).

   ◦ `address`: адрес клиента (IP и порт).

   ◦ `thread_id`: ID потока, который обслуживает этого клиента.

### Функции работы с клиентами и базой

• `broadcast_message`: Рассылает сообщение всем клиентам, кроме одного. Проходит по массиву clients и отправляет сообщение каждому.
    
• `remove_car(brand)`: Удаляет машину из базы.
        1. Блокирует мьютекс, чтобы другие потоки не мешали.
        2. Если находит машину — сдвигает все последующие элементы массива на одну позицию влево, удаляет элемент и уменьшает счетчик.
        3. Разблокирует мьютекс и возвращает 1 (успех) или 0 (не найдена).

• `add_car(brand)`: Добавляет машину в базу.
        1. Блокирует мьютекс.
        2. Если есть место, копирует название в конец массива и увеличивает счетчик.

• `send_car_list(client_socket)`: Отправляет конкретному клиенту список всех машин.
        1. Блокирует мьютекс базы (чтобы список не изменился во время отправки).
        2. Формирует сообщение и отправляет его по сокету клиента.
           
• `handle_client`: Выполняется в отдельном потоке для КАЖДОГО подключившегося клиента.
        1. Бесконечный цикл: пока сервер работает.
            ▪ Ждет, пока от клиента придет сообщение. Печатает полученное сообщение в консоль сервера.
            ▪ Анализирует команду
		и если машина не найдена или неизвестная команда, то уведомляет клиента
        2. Завершение клиента: Когда клиент отключается, поток удаляет его из глобального массива clients (под защитой clients_mutex), закрывает сокет и освобождает память.

•  `shutdown_server`: Обработка сигналов
          по Ctrl+C или SIGTERM.

• `main`: Запуск сервера и работа




## client.c (Клиент автосалона)

Это программа, через которую пользователь общается с сервером.

Обрабатываются сигналы, доступны команды BUY, SELL, exit, CARS

• `listener`: Поток для прослушивания

• `main`: Запуск клиента
	С IP-адресом и портом сервера
	Создается сокет, подключается к серверу по указанным адрресу и порту, выводит меню, создает поток-слушатель
